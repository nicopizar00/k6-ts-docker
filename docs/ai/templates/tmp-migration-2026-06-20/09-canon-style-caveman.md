# 09 · Caveman canon · doc style · emoji 例外（canon / style）

> ⚠ IDEA scratch。Output-efficiency canon vs persisted-doc rules。

Caveman canon 與文風,易漏：

- **Caveman canon 源**:`punch-build-caveman/SKILL.md` 單一源,per-phase 深度(lite/full/ultra)。考:**此 skill 不在最小範本**!採全 caveman 治理須另採。半採 → 無 canon。
- **output-only**:Caveman canon 治**輸出**,非 persisted 文。考:採者誤壓持久文成 caveman/wenyan?canon 禁之(persistent artifacts)。
- **wenyan 禁於持久**:docs/ADR/context-map/skill/prompt/registry 禁文言。考:範本 assets 自身文言 —— 例外(export template)。採後活配置勿續文言!轉英/本地。易漏之矛盾。
- **emoji 例外**:`/punch-document` 持久文許 emoji(✅⚠️❌)助掃,為 Caveman no-emoji 之 carve-out。考:採者知此例外否?否則 emoji 被 caveman 規誤刪。
- **per-phase 深度表**:Document/Spec lite、Plan/Review/Ship full、Build/Test ultra。考:單源表(ADR 0003 Rev 3),採後勿復述、勿漂。
- **evidence verbatim**:憑(exit code、readiness)逐字引,不壓。考:採者壓 evidence?canon 禁。
- **caveman-wenyan 雙壓**:此遷移夾即雙壓。考:disposable scratch 可;持久文不可。勿混。
- **terminal vs persisted**:ultra 唯 terminal/status summary。考:採者於持久報用 ultra?越界。
- **mode 切換**:`/caveman lite|full|ultra`、"stop caveman"。考:採環境支援 caveman skill 否?無 skill 則 canon 唯文字,無強制。

> ⚠ Miss → 採後續寫文言持久文,違 canon;或 emoji 例外漏,掃性降。
